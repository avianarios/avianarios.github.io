Empaquetadores

A grandes rasgos, el empaquetamiento de módulos (module bundling) es simplemente el proceso de tomar un grupo de módulos, y sus dependencias, y juntarlos en uno o más ficheros, en el orden correcto pero, como todo en el desarrollo web, el diablo está en los detalles...

¿Por qué relizarlo?

Cuando se divide un programa en módulo, se organizan en ficheros y directorios. También es bastante probable que se tenga un grupo de módulos para las bibliotecas que se estén usando, como Underscore o React.

El resultado es cada uno de esos ficheros debe ser incluido en el HTML en una etiqueta "script", lo cual hace que el navegador lo cargue cuando el usuario visite la página HTML. Con este esquema, el navegador debe cargar cada fichero individualmente, lo cual ralentiza el proceso de carga.

Para resolver este problema, se empaquetan o concatenan todos los ficheros que se incluirían por separado, en uno grande (o en varios) para reducir el número de peticiones al servidor. Otro enfoque para reducir el tiempo de carga es minimizar (minify) el código empaquetado. Consiste en eliminar caracteres innecesarios del código fuente (espacios en blanco, comentarios, marcas de nueva línea, etc.) para reducir el tamaño del código sin cambiar la funcionalidad.

Webpack

Webpack es un empaquetador de código abierto para JavaScript, HTML, CSS o imágenes.
